# Backend: экспертный анализ интерактивных изданий

Прототип backend-сервиса для магистерской диссертации. Сервис принимает ссылку на интерактивное издание, извлекает признаки страницы, оценивает их по методической рубрике и, при наличии `OPENAI_API_KEY`, формирует экспертный отчет через OpenAI Responses API.

Основной запуск сделан без внешних зависимостей: сервер работает на стандартной библиотеке Python. Это важно для прототипа магистерской диссертации: backend можно показать даже без настройки FastAPI, Docker или отдельной инфраструктуры.

## Почему такая архитектура

Модель не обучается с нуля. Для магистерской диссертации реалистичнее и надежнее использовать связку:

1. автоматический сбор признаков страницы;
2. методическая рубрика из пособия;
3. сменный ИИ-провайдер для связного экспертного отчета.

По умолчанию проект запускается с `AI_PROVIDER=none`: базовый анализ и методический отчет работают без API-ключей. Для расширенного ИИ-отчета можно подключить OpenAI, Google Gemini или локальный Ollama.

## Возможности MVP

- `POST /api/analyze` - анализ страницы по URL.
- `GET /api/rubric` - список критериев оценки.
- `GET /health` - проверка состояния сервиса.
- Безопасная URL-валидация: по умолчанию запрещены `localhost`, приватные IP и не-HTTP(S) схемы.
- Детерминированный отчет без API-ключа.
- AI-отчет через сменный провайдер: `openai`, `gemini` или `ollama`.

## API для frontend

Подробный контракт API описан в [docs/API.md](docs/API.md).

Для быстрой интеграции:

- base URL локально: `http://127.0.0.1:8000`
- основной endpoint: `POST /api/analyze`
- обязательное поле запроса: `url`
- если `include_features=false`, поле `features` вернется как `null`
- если AI недоступен или `AI_PROVIDER=none`, backend все равно вернет методический отчет, а `ai.used=false`

OpenAPI-описание лежит в [docs/openapi.json](docs/openapi.json).

## Быстрый старт

Зависимости для базового запуска не нужны. Достаточно Python 3.11+.

```powershell
Copy-Item .env.example .env
```

В `.env` укажите:

```env
AI_PROVIDER=none
```

Для облачного ИИ-отчета можно выбрать провайдера:

```env
AI_PROVIDER=openai
OPENAI_API_KEY=sk-...
OPENAI_MODEL=gpt-5.4-mini
```

```env
AI_PROVIDER=gemini
GEMINI_API_KEY=...
GEMINI_MODEL=gemini-2.5-flash
```

Для полностью локальной демонстрации без внешнего API:

```env
AI_PROVIDER=ollama
OLLAMA_BASE_URL=http://127.0.0.1:11434
OLLAMA_MODEL=qwen2.5:7b
```

Запуск стандартного backend:

```powershell
python -m app.server --host 127.0.0.1 --port 8000
```

Пример запроса:

```powershell
Invoke-RestMethod `
  -Uri http://127.0.0.1:8000/api/analyze `
  -Method POST `
  -ContentType 'application/json' `
  -Body '{"url":"https://example.com","use_ai":false}'
```

## Формат запроса

```json
{
  "url": "https://example.com",
  "use_ai": true,
  "include_features": true
}
```

Если `use_ai=true`, но выбранный провайдер не настроен, backend вернет обычный методический отчет и поле `ai.used=false`.

## Дополнительный FastAPI-вариант

В проекте также есть `app/main.py` для запуска через FastAPI:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

Если на машине установлен экспериментальный Python 3.14 и FastAPI/Pydantic падают на импорте, используйте основной запуск `python -m app.server`. Он не зависит от этих пакетов.

## Ограничения прототипа

- Без Playwright backend анализирует HTML, который вернул сервер, а не полностью отрендеренное JavaScript-состояние.
- Для полноценной оценки интерактивных лонгридов следующим шагом стоит добавить Playwright-снимки desktop/mobile, Lighthouse и axe-core.
- Сервис не заменяет экспертную оценку, а автоматизирует первичный аудит и генерацию отчета по методике.
